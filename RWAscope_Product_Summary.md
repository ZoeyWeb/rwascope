# RWAscope — 香港RWA合规情报平台

**首个为香港虚拟资产中心量身打造的RWA市场透明度解决方案**

> 面向监管机构、机构投资者、资产发行方，通过 **RARM（RWA Asset Risk Matrix）六层框架** 提供结构化合规风险情报和尽调工具。
>
> **法律声明** — RWAscope 提供学术研究工具和市场情报，不提供信贷评级、投资建议或受监管的金融服务。详见 [COMPLIANCE.md](./COMPLIANCE.md)。

最后更新：2026-05-12（HK视角重构版本）

## 一、为什么是现在，为什么是RWAscope

### 1.1 HK的RWA机遇窗口

**政策驱动**
- **EnsembleTX（Project Ensemble）**：HKMA于2025年11月启动真实价值tokenized deposit结算试点，2026年全年运行，参与机构包括HSBC、渣打、中银、贝莱德、富兰克林邓普顿——RWA结算是核心应用场景
- **e-HKD批发优先**：HKMA已完成e-HKD Phase 2（2025年10月），结论明确：**当前优先推进批发（Wholesale）场景**，支持tokenized deposit与RWA跨行结算；零售普及待条件成熟后再议
- **稳定币监管落地**：Stablecoins Ordinance于2025年8月1日生效；HSBC与Anchorpoint Financial于2026年4月获首批HKD稳定币牌照，直接连接RWA结算基础设施
- **虚拟资产中心建设**：HK Policy Statement 2.0（2025年6月）明确"可持续金融中心+虚拟资产枢纽"战略，RWA tokenization被列为核心支柱（LEAP框架）
- **SFC持牌交易平台**：截至2026年2月，SFC已批准12家VATP，持牌平台可分发tokenized securities，为RWA二级市场提供基础
- **跨境人民币tokenization**：大湾区合作，HK作为离岸RMB tokenization的核心窗口

**市场缺口**
- 传统DeFi仪表盘（如DeFiLlama）只展示TVL和价格，无法回答：
  - *"这个RWA协议在HK法规下合规吗？"*
  - *"托管安全性如何？能真正赎回吗？"*
  - *"与同类项目相比，风险在哪？"*
- HK机构投资者缺少**结构化合规尽调工具**
- 监管机构缺少**市场成熟度实时监测**手段

### 1.2 RWAscope的独特定位

**不是评级机构**（避开CRA监管）  
**不是投资顾问**（避开SFC监管）  
**而是：**
- 📊 **市场情报平台** — 实时聚合HK RWA生态数据
- 🔍 **研究工具包** — 提供RARM框架让用户自主尽调
- 🌐 **政策智能** — 自动追踪HKMA/SFC最新规定，预测市场影响
- 🏦 **生态地图** — 展示HK的托管机构、律师行、审计方、基金的RWA参与度

**为什么会成为"第一"**
- 首家专注**HK + RWA**的信息平台（非全球通用）
- 框架由HKUST博士研究（SARM/RARM发表于Digital Finance）
- 与HKMA/SFC的监管框架**天然对齐**（L1法规层对应真实执法）
- 构建**市场共识**：HK机构、律师、发行方逐步依赖这个框架做决策

## 二、核心价值主张

### 问题
```
HK机构投资者配置RWA时：
  ❌ 缺少合规系统性评估（靠人工阅读文件+律师咨询）
  ❌ 无法快速对标（A项目vs B项目的风险差异在哪？）
  ❌ 缺少政策跟踪（HKMA新规出台，影响哪些项目？）

HK监管机构：
  ❌ 无实时市场风险监测（只能看事后报告）
  ❌ 无法评估HK RWA生态的整体健康度
  ❌ 缺少政策影响预测工具

HK RWA发行方：
  ❌ 不知道机构投资者看重什么（合规? 流动性? 托管?)
  ❌ 无法对标竞品的短板
  ❌ 缺少改进方向的优先级指导
```

### 解决方案
```
RWAscope = 透明度 + 结构化 + 本地化

✅ 透明度
   市场一览：DeFiLlama追踪100+ RWA协议实时对比（链上数据 + RARM评估框架）
   全球RWA TVL：$17B+（2026年5月，DeFiLlama数据）
   
✅ 结构化
   RARM六层框架：法规 → 估值 → 托管 → KYC → 流动性 → 结算
   每层5-8个子指标，避免"黑箱评分"
   
✅ 本地化
   HKMA政策追踪：新规发布24小时推送市场影响评估
   HK生态地图：谁是主要托管方？谁能通过审计？
   HK RWA专库：首先覆盖所有HK相关项目
```

## 四、HK特色与差异化

### 4.1 为什么现在聚焦HK

**全球RWA平台很多，但没有"HK优先"的**

**现有主要竞争平台：**

- **RWA.xyz**：目前全球最主要的专门RWA数据平台，追踪tokenized treasuries、私人信贷、房产、商品，提供TVL实时排名和协议统计——但**无HK政策追踪，无结构化合规评估框架**
- **DeFiLlama**：全球化TVL数据聚合，覆盖RWA category，但**无合规维度，无政策层**
- **Token Terminal**：专注协议财务指标（收入、费用、估值），**无合规框架**
- **Messari**：定期发布RWA赛道研究报告，但**通用框架，无HK本地视角**
- **Security Token Market**：安全代币数据，**侧重美国SEC监管框架**

**RWAscope的不同：HK政策对齐 + 合规框架**
- 📍 L1法规层**直接对标HKMA/SFC**指引（不是SEC框架，不是MiCA）
- 🏦 L3托管层优先列举**HK认可的托管机构**（汇丰、DBS、JP Morgan HK、Paxos Trust）
- 🇭🇰 Policy Tracker**只关注HK相关政策变化**（RWA.xyz、DeFiLlama均无此功能）
- 🔍 RARM六层框架提供**结构化合规尽调**（RWA.xyz只有数据，无评估框架）
- 🤝 生态地图**映射HK律师行、审计方、基金**的参与度

   需要进一步核证"。决定权完全在用户。

❌ 不提供托管保障
   
   展示某RWA采用Paxos托管不代表风险消除，

## 附录：术语表

| 术语 | 解释 |
|------|------|
| **RWA** | Real World Asset，真实世界资产的区块链代币化 |
| **RARM** | RWA Asset Risk Matrix，我们的六层风险评估框架 |
| **L1-L6** | 法规、估值、托管、KYC、流动性、结算的六个层级 |
| **赎回窗口** | 投资者请求转换为现金的周期（T+1/T+3/T+30等） |
| **NAV** | Net Asset Value，净资产价值，RWA的公平价值 |
| **托管方** | 持有实物资产的第三方金融机构（如Paxos Trust Company，注意区别于Paxos Gold这一RWA协议产品） |
| **预言机** | 提供链外数据上链的服务（如Chainlink） |
| **多签** | Multi-signature，需多个私钥才能执行的智能合约操作 |
| **AML/CFT** | Anti-Money Laundering / Combating Terrorist Financing，反洗钱监管 |
| **e-HKD** | HKMA的数字港元（CBDC），当前优先推进批发（Wholesale）场景 |
| **EnsembleTX** | HKMA Project Ensemble的试点阶段（2026年全年），支持tokenized deposit真实价值结算 |
| **VATP** | Virtual Asset Trading Platform，SFC持牌虚拟资产交易平台，截至2026年2月共12家获批 |
| **Stablecoins Ordinance** | 香港稳定币条例，2025年8月1日生效，HSBC与Anchorpoint于2026年4月获首批牌照 |
| **SFC** | 香港证券及期货事务监察委员会 |
| **HKMA** | 香港金融管理局 |
| **LEAP框架** | HK政府数字资产监管路线图，涵盖RWA法律审查、tokenized bond正规化、ETF tokenization等 |
